# Guide de déploiement PATHWAY Assessment
## GitHub Pages — Compte : racime95

---

## ÉTAPE 1 — Créer le dépôt

1. Connecte-toi sur **github.com** avec ton compte `racime95`
2. Clique sur le bouton vert **"New"** (en haut à gauche)
3. Dans le champ **"Repository name"**, tape exactement :
   ```
   pathway-assessment
   ```
4. Laisse tout le reste par défaut (Public, pas de README)
5. Clique sur le bouton vert **"Create repository"**

---

## ÉTAPE 2 — Uploader les 3 fichiers HTML

1. Sur la page de ton nouveau dépôt, clique sur **"uploading an existing file"**
   (le lien est au milieu de la page, texte en bleu)
2. Glisse-dépose les 3 fichiers HTML dans la zone pointillée :
   - `PATHWAY_U13_MSQ99.html`
   - `PATHWAY_U14_ACSI28.html`
   - `PATHWAY_U15_MTQ48.html`
3. En bas de la page, clique sur le bouton vert **"Commit changes"**
4. Attends 5 secondes que l'upload se termine

---

## ÉTAPE 3 — Activer GitHub Pages

1. Clique sur l'onglet **"Settings"** (en haut du dépôt, icône engrenage)
2. Dans le menu gauche, clique sur **"Pages"**
3. Sous **"Branch"**, clique sur le menu déroulant qui dit **"None"**
4. Sélectionne **"main"**
5. Clique sur **"Save"**
6. Attends **2 à 3 minutes** — GitHub génère ton site

---

## ÉTAPE 4 — Vérifier que ça fonctionne

Ouvre ce lien dans ton navigateur :
```
https://racime95.github.io/pathway-assessment/PATHWAY_U13_MSQ99.html
```

Si la page PATHWAY s'affiche → tout est bon.

## En cas de problème

**Le lien ne s'ouvre pas** → Attendre encore 2-3 minutes après l'étape 3, GitHub Pages prend parfois 5 minutes à la première activation.

**L'interface est blanche** → Vider le cache du navigateur (Ctrl+Shift+R sur Windows, Cmd+Shift+R sur Mac).

**Un coach perd ses données** → Chaque appareil garde ses propres données. Exporter le JSON après chaque session est obligatoire pour centraliser les données.
